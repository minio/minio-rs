# What is Pending in minio-rs to Comply with Recent minio-go Changes

> Generated by comparing recent `minio-go` feature commits (window: `#2123`/`bc9e303` → `#2233` RDMA, synced at both HEADs) against the `minio-rs` codebase. Each item was investigated by reading the actual `git show` diff in minio-go and the corresponding Rust code, then adversarially re-verified against minio-rs source. Go-only noise (dep bumps, ini/yaml/json renames, build-tag and goroutine-leak fixes) was excluded.

## Summary

Of 24 tracked minio-go changes, **18 are pending** in minio-rs and **6 are already compliant or not applicable**. The pending work splits into **10 fixes/additions to existing APIs** (1 missing, 9 partial), **6 net-new APIs** (all missing), and **2 infrastructure items** (1 missing, 1 partial). Most fixes are small/medium effort; the two large items are the Inventory API and the LDAP STS credential provider, both of which require new subsystems.

## Pending — fixes/additions to existing APIs

- **New AWS checksum algorithms** (`checksums-new`, partial, **medium**) — go `f29e568 (#2221)`, `88ad5c9 (#2123)`. Missing 5 algorithms (MD5, SHA512, XXHash64, XXHash3, XXHash128) from the `ChecksumAlgorithm` enum (`src/s3/utils.rs:186-192`), its `as_str`/`header_name`/`FromStr` methods, `compute_checksum()` (`utils.rs:319-423`), and HTTP header constants (`src/s3/types/header_constants.rs:111-119`). XXHash3/XXHash128 require new crate deps. CopyObject checksum selection already works.

- **`x-minio-source-time` header override** (`source-time-header`, partial, **small**) — go `68e87b1 (#2219)`. `StatObjectResponse.last_modified()` (`src/s3/response/stat_object.rs:64-69`) reads only `Last-Modified` with no `X-Minio-Source-Mtime` override; header constant undefined in `header_constants.rs`; `GetObjectResponse` (`src/s3/response/get_object.rs`) has no `last_modified()` accessor at all.

- **ChecksumAlgorithm in list responses** (`list-checksum-algo`, partial, **small**) — go `570a610 (#2211)`. `ListEntry` struct (`src/s3/types/basic_types.rs:26-41`) lacks a `checksum_algorithm` field; `parse_list_objects_contents()` (`src/s3/response/list_objects.rs:132-147`) does not extract `ChecksumAlgorithm` from XML.

- **Honor StartAfter when listing versions** (`list-versions-startafter`, partial, **small**) — go `30849ad (#2212)`. `ListObjectVersions::from()` (`src/s3/builders/list_objects.rs:451-468`) copies only `key_marker`/`version_id_marker`, not `start_after`; the struct lacks a `start_after` field and `to_s3request` never emits `start-after`.

- **SignV4WithServiceType public function** (`signv4-servicetype`, **missing**, **small**) — go `2eacfe9 (#2214)`. Internal `sign_v4()` (`src/s3/signer.rs:287`) takes a `service_name` param but is private; all public signers (`sign_v4_s3:322`, `presign_v4:352`, etc.) are `pub(crate)` and hardcode `"s3"`. No public equivalent exists.

- **Content-Encoding accessor** (`content-encoding`, partial, **small**) — go `4976cb3 (#2208)`. `CONTENT_ENCODING` constant (`header_constants.rs:127`) is used only request-side; neither `GetObjectResponse` nor `StatObjectResponse` exposes a typed `content_encoding()` accessor (should trim whitespace per Go).

- **Max object size ~48.83 TiB** (`max-object-size`, partial, **small**) — go `5b6d388 (#2190)`. `MAX_OBJECT_SIZE = 1024 * MAX_PART_SIZE` (5 TiB) at `src/s3/builders/put_object.rs:957` instead of `MAX_PART_SIZE * MAX_MULTIPART_COUNT`; enforced at `put_object.rs:982-986` and `client/mod.rs:595-597`; error message at `error.rs:103` reflects old limit; `ComposeObject` (`copy_object.rs:749-778`) has no `part_size` field.

- **AppendObject post-call checks** (`append-postcall-checks`, partial, **small**) — go `8aa3271 (#2178)`. No validation that `x-amz-object-size` header is present (`HasObjectSize` at `response_traits.rs:231-237` silently returns 0); no check that `final_size == offset + original_size` (`send_mpa` at `builders/append_object.rs:320` assigns without comparison). Checks exist only in example code.

- **Omit empty lifecycle Filter XML** (`lifecycle-filter-xml`, partial, **small**) — go `b392776 (#2177)`. `Filter` (`src/s3/types/notification/filter.rs:99-140`) always emits `<Filter></Filter>`; no `is_null()`/`is_empty()` guard; `LifecycleConfig::to_xml()` (`lifecycle_config.rs:81`) calls `rule.filter.to_xml()` unconditionally.

- **Conditional PUT (If-Match / If-None-Match)** (`conditional-put`, partial, **medium**) — go `754a055 (#2153)`. `UploadPart`, `PutObjectContent`, and `CompleteMultipartUpload` (`src/s3/builders/put_object.rs:326-392, 534-604, 197-223`) lack typed `match_etag`/`not_match_etag` fields and the Go quoting/`*`-wildcard logic, unlike GetObject/StatObject/CopyObject. Injectable today only via raw `extra_headers`.

## Pending — net-new APIs/features

- **GetObjectAttributes** (`getobjectattributes-checksum-type`, **medium**) — go `d457cd4 (#2217)`. Entire API absent (no client method, builder, or `ObjectAttributesResponse`). Go added `ChecksumCRC64NVME`/`ChecksumType` fields plus `ChecksumMap()`/`ChecksumMode()`/`ChecksumType()` helpers and COMPOSITE/FULL_OBJECT mode constants. `ChecksumAlgorithm::CRC64NVME` primitive already exists.

- **UpdateObjectEncryption** (`update-object-encryption`, **medium**) — go `ffb7ec0 (#2204)`, `9ab4afb (#2215)`. No builder, client method, or response type. Needs `?encryption` PUT with XML body accepting `KMSKeyArn`/`BucketKeyEnabled`/`VersionID`, returning a `VersionID` from the `x-amz-version-id` header.

- **GCS bulk delete fallback** (`gcs-bulk-delete`, **medium**) — go `23199ac (#2185)`. Needs GCS endpoint detection (no `is_google_endpoint()` on `BaseUrl`) and a single-`DELETE`-per-object fallback path; `DeleteObjects`/`DeleteObjectsStreaming` (`src/s3/builders/delete_objects.rs`) unconditionally use the bulk POST `?delete` API.

- **QoS APIs** (`qos-api`, **medium**) — go `ec35de6 (#2148)`. Missing `GetBucketQOS`/`SetBucketQOS`/`GetBucketQOSMetrics` and types (`QOSConfig`, `QOSRule`, `QOSMetric`, `CounterMetric`, `QOSNodeStats`). **Blocker:** requires YAML serialization; `Cargo.toml` has no `serde_yaml` (existing config APIs are XML/JSON only).

- **Inventory APIs** (`inventory-api`, **large**) — go `f64cdbd (#2147)` through `9207380 (#2172)`. Entire subsystem missing: ~7 client methods (Generate/Put/Get/Delete/List/ListIterator/JobStatus) plus `InventoryConfiguration`, `InventoryListResult`, and `InventoryJobStatus` (28 fields). No `minio-inventory` query-param plumbing.

- **LDAP STS provider + ConfigName** (`ldap-sts-configname`, **large**) — go `6217ce2 (#2173)`. Entire LDAP STS identity provider absent; `src/s3/creds.rs` has only the `Provider` trait and `StaticProvider`, no HTTP-based async credential fetching. `ConfigName` is moot until the provider exists. (`InvalidIdpConfigName` error variant exists but is unused.)

## Pending — infrastructure/plumbing

- **S3 Outposts support** (`s3-outposts`, **missing**, **medium**) — go `d75e965 (#2201)`. Zero Outposts references in the codebase. Missing: Outposts endpoint-detection regex (`src/s3/http.rs`), `s3-outposts` service-type constant, region extraction/validation from Outposts hosts, public `SignV4Outposts`/`PreSignV4Outposts`/`SignV4TrailerOutposts`/`StreamingSignV4Outposts` variants (all public signers hardcode `"s3"`), `service_type` on `ChunkSigningContext`, client-init endpoint detection with signer-type override, and presign `x-amz-s3session-token` handling.

- **HTTP 200 + ErrorResponse XML body** (`200ok-error`, partial, **medium**) — go `ec775e6 (#2139)`. No opt-in flag (`expect200OKWithError` equivalent) on `S3Request` (`src/s3/types/s3_request.rs:28-78`); `execute_internal` (`client/mod.rs:852`) returns `Ok(resp)` on any success without inspecting the body; **`DeleteObjectsResponse::result()` (`src/s3/response/delete_object.rs:119`) will panic via `assert_eq!` on a root-level `Error` element** (e.g., `SlowDownWrite`) rather than producing a retryable error. Per-object `Error` children are handled (hence partial).

## Already compliant / not applicable

- **empty-stream-putobject** (present) — Rust `read_upto` never returns an EOF error; empty streams route to single-part upload. Go bug `c44cb2a (#2207)` structurally cannot occur.
- **readat-seek** (N/A) — no stateful `Read`/`ReadAt`/`Seek` reader; each GetObject is an independent Range request, so Go's shared-offset bug `1451162 (#2197)` has no analogue.
- **region-ap-east-2** (present) — regions are inserted generically (`http.rs:405-416`), no per-region table; `ap-east-2` works with no code change.
- **ipv6-brackets** (present) — `BaseUrl::from_str` (`http.rs:292-295`) brackets only IPv6 via `parse::<Ipv6Addr>()`, verified by tests; equivalent to Go's `ip.To4() == nil`.
- **tags-json** (N/A) — tags are `HashMap<String,String>` everywhere, serializing to flat JSON natively; Go's XML-leak fix has no analogue.
- **non-dualstack-regions** (N/A) — no static endpoint map; ISO TLDs fail the `amazonaws.com` regex and are used verbatim, so fabricating a bad dualstack ISO endpoint is structurally impossible.
